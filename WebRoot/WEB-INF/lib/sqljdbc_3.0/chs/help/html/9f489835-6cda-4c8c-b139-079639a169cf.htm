setResponseBuffering 方法 (ISQLServerStatement)

将此 SQLServerStatement 对象的响应缓冲模式设置为不区分大小写的 String full 或 adaptive。

语法

public final void setResponseBuffering(java.lang.String value)

参数

value

包含响应缓冲模式的 String。有效模式可以为下列不区分大小写的字符串之一：full 或 adaptive。

异常

SQLServerException

备注

adaptive 指定在必要时缓冲尽可能少的数据。

full 指定在运行时从服务器读取全部结果。

adaptive 是 JDBC Driver 2.0 版和 3.0 版中的默认值。full 是 JDBC Driver 2.0 之前版本中的默认值。

setResponseBuffering 方法允许覆盖当前 SQLServerStatement 对象的 responseBuffering 连接 String 属性。有关使用响应缓冲模式的更多信息，请参阅使用自适应缓冲。

如果应用程序将 setResponseBuffering 方法设置为一个无效的参数值，则会引发 SQLServerException。
